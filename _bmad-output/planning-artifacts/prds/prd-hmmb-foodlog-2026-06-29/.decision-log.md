# Decision Log — PRD hmmb-foodlog UX/UI

## 2026-06-29 — Finalizado (status: final)

PRD finalizado após polish editorial. Glossário expandido com Revisão, Aceitar e Padrão alimentar. Sem decisões novas em relação ao draft.

## 2026-06-29 — Iniciado

### DEC-001: Escopo frontend-only
**Decisão:** Nenhuma alteração de backend (migrations, rotas, endpoints).
**Motivo:** Todos os dados necessários estão disponíveis via APIs existentes. Escopo cirúrgico preserva estabilidade do backend.
**Registrado em:** §5 Non-Goals

### DEC-002: Nova aba Painel na segunda posição do tab bar
**Decisão:** Tab bar: Revisão | Painel | Tags | Compartilhar | Relatório.
**Motivo:** Hugo confirmou que quer tanto mini-resumo na Revisão quanto uma aba dedicada para padrões.
**Registrado em:** §4.2 FR-5

### DEC-003: Auditoria removida do tab bar principal
**Decisão:** Aba Auditoria acessível via mecanismo secundário (URL ou link no rodapé), não via tab bar.
**Motivo:** Hugo confirmou que é ferramenta de suporte de desenvolvimento, não uso diário.
**Registrado em:** §4.2 FR-4

### DEC-004: Parede de Fotos como vista primária do Painel
**Decisão:** Grade de fotos com sobreposição de hora + kcal é a vista padrão do Painel.
**Motivo:** Hugo solicitou explicitamente uma vista onde as fotos sejam o destaque para identificação visual de padrões alimentares.
**Registrado em:** §4.4 FR-9

### DEC-005: Vista Compartilhada com múltiplas vistas para nutricionista
**Decisão:** Adicionar Timeline e Resumo do Período à vista compartilhada (em v2).
**Motivo:** Hugo confirmou que quer que o nutricionista tenha múltiplas formas de analisar os dados.
**Registrado em:** §4.6 FR-15 / FR-16

### DEC-006: Confiança via borda esquerda colorida (não badge de %)
**Decisão:** Confiança da IA exibida como borda esquerda de 4px no card, não como percentual de texto.
**Motivo:** Reduz ruído visual; o percentual bruto é técnico demais para o fluxo de revisão diária.
**Registrado em:** §4.5 FR-13

### DEC-007: FR-11 e FR-15-16 fora do MVP
**Decisão:** Gráfico de barras e melhorias da vista compartilhada entram em v2.
**Motivo:** O gráfico SVG puro tem complexidade considerável para um único caso de uso; a vista compartilhada atual ainda funciona. MVP deve resolver primeiro o ponto de dor principal (visão de padrões pessoal).
**Registrado em:** §6.2

### DEC-008: FR-17 (melhorias no calendário compartilhado) como candidato ao MVP
**Decisão:** FR-17 está fora do MVP mas com nota de que pode ser incluído se o Painel avançar antes do previsto.
**Motivo:** Implementação simples (apenas CSS); baixo risco de incluir.
**Registrado em:** §6.2 [NOTE FOR PM]
